Customer Satisfaction in Sephora Products

This project investigates how customer satisfaction can be predicted using product metadata, review text, Aspect-Based Sentiment Analysis (ABSA), feature engineering, and machine learning techniques.

By combining structured product information with sentiment-based textual features, the project aims to better understand customer behavior and personalization patterns in beauty product experiences.

Dataset

Sephora Products & Reviews Dataset

The project uses the Sephora Products and Skincare Reviews dataset obtained from Kaggle.

The dataset combines structured product information, customer-related variables, and unstructured textual review data collected from Sephora’s online platform.

The structured variables include:

  • Brand information
  • Product categories
  • Product prices
  • Ingredients
  • Product highlights
  • Product ratings

In addition, the dataset contains customer characteristics such as skin type, skin tone, and hair color.

The dataset contains more than 8,000 products and approximately 1 million reviews, making it highly suitable for sentiment analysis and machine learning applications.

Pipeline

Preprocessing & Feature Engineering

Before model training, the raw dataset undergoes several preprocessing and transformation stages.

1. Data Loading & Merging

Review files are merged into a unified dataframe structure and standardized using product-based joins.

2. Target Variable Construction

Ratings are transformed into a binary classification problem: Negative/Neutral and Positive.

3. Data Cleaning

Missing review texts, invalid scores, and inconsistent text values are cleaned during preprocessing.

4. Balanced Subsampling

Stratified balanced sampling is applied to reduce computational cost while preserving class distribution.

5. Aspect-Based Sentiment Analysis (ABSA)

The project uses the Hugging Face model Yangheng/deberta-v3-base-absa-v1.1 to separately analyze skin, hair, and general product sentiment.

6. Feature Engineering

Additional features include ingredient indicators, vegan labels, review length, and user-product compatibility variables.

7. Class Imbalance Management

Controlled undersampling and stratified train-test splitting are used to improve model fairness and evaluation reliability.
