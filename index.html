Dataset Description

The dataset used in this project is the Sephora Products and Skincare Reviews dataset obtained from Kaggle. It contains detailed information about skincare products and customer reviews collected from Sephora’s online platform.

The dataset combines:

  • Structured product information
  • Customer-related attributes
  • Unstructured textual review data

The structured variables include brand information, product categories, prices, ingredients, product highlights, and product ratings.

In addition, the dataset contains user characteristics such as skin type, skin tone, and hair color.

The dataset contains over 8,000 products and approximately 1 million customer reviews, making it highly suitable for machine learning and sentiment analysis applications.

Preprocessing Pipeline

1. Data Loading and Merging

Review CSV files are merged into a unified dataframe structure. Corrupted rows are cleaned, rating columns are standardized, and review/product datasets are merged using product_id.

2. Target Variable Construction

Product ratings are transformed into a binary classification problem:

  • 1–3 stars → Negative / Neutral
  • 4–5 stars → Positive

3. Data Cleaning

Missing review texts are removed, invalid scores are filtered, and inconsistent text values are corrected.

4. Balanced Subsampling

A balanced subsampling strategy is applied to reduce computational cost while preserving the overall class distribution.

5. Aspect-Based Sentiment Analysis (ABSA)

The project uses the Hugging Face ABSA model:

Yangheng/deberta-v3-base-absa-v1.1

The model separately analyzes:

  • Skin sentiment
  • Hair sentiment
  • General product sentiment

6. Feature Engineering

Additional engineered variables include review length, ingredient-based features, vegan indicators, cruelty-free indicators, and user-product compatibility variables.

7. Class Imbalance Management

Controlled undersampling and stratified train-test splitting are used to reduce majority-class bias and improve model fairness.
